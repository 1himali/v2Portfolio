Hospitality / SaaS

Property Management System (PMS)

A full-stack Property Management System built from scratch — covering rooms, dining, billing, inventory, payroll, and analytics for independent hospitality businesses.

Problem

Independent hospitality businesses — B&Bs, hostels, small hotels — operate across fragmented tools: manual registers for rooms, separate POS for restaurant, spreadsheets for inventory and payroll, and no unified analytics view. Enterprise PMS solutions are priced for large hotels.

Solution

A unified Spring Boot application serving a REST API documented via Swagger/OpenAPI, a static browser UI with role-gated navigation, PostgreSQL with 15 Flyway migrations, JWT with four role tiers, and integrated modules: rooms, restaurant, billing, inventory, payroll, analytics, activity log, and AI assistant.

Outcome

  • 85 automated tests across 12 test classes validate system correctness
  • 15 Flyway database migrations provide a complete, reproducible schema history
  • Four-role access control enforced server-side across every API endpoint
  • Swagger UI generates live API documentation enabling third-party developer onboarding
  • Mock mode allows safe testing of third-party integrations without live API calls
  • A B&B with 10-15 rooms could replace 3-4 separate tools with this single system

Technologies

Java 21 Spring Boot 3.5 PostgreSQL Flyway JWT Swagger JUnit 5 Testcontainers
